# Comparison Operators in Python

**Comparison operators** compare two values and return a Boolean result (`True` or `False`).

They are commonly used in:

* `if` statements
* `while` loops
* Filtering data
* Machine Learning logic
* Data validation

---

# 1. Equal To (`==`)

Checks whether two values are equal.

```python
x = 10
y = 10

print(x == y)
```

Output:

```text
True
```

Example:

```python
name = "Someshwar"

print(name == "Someshwar")
```

Output:

```text
True
```

---

# 2. Not Equal To (`!=`)

Checks whether two values are different.

```python
x = 10
y = 20

print(x != y)
```

Output:

```text
True
```

---

# 3. Greater Than (`>`)

```python
salary = 100000

print(salary > 50000)
```

Output:

```text
True
```

---

# 4. Less Than (`<`)

```python
age = 15

print(age < 18)
```

Output:

```text
True
```

---

# 5. Greater Than or Equal To (`>=`)

```python
score = 75

print(score >= 75)
```

Output:

```text
True
```

---

# 6. Less Than or Equal To (`<=`)

```python
temperature = 20

print(temperature <= 25)
```

Output:

```text
True
```

# Comparison with Strings

Python compares strings lexicographically (dictionary order).

```python
print("Apple" == "Apple")
```

Output:

```text
True
```

```python
print("Apple" < "Banana")
```

Output:

```text
True
```

Because `"A"` comes before `"B"`.

---

# Case Sensitivity

```python
print("python" == "Python")
```

Output:

```text
False
```

Python string comparison is case-sensitive.

To compare ignoring case:

```python
print("python".lower() == "Python".lower())
```

Output:

```text
True
```

# Comparison with Lists

```python
list1 = [1, 2, 3]
```


# Chained Comparisons

Python supports chained comparisons.

```python
age = 25

print(18 <= age <= 60)
```

Output:

```text
True
```

Equivalent to:

```python
print(age >= 18 and age <= 60)
```

# Identity Operators (`is` and `is not`)

A common source of confusion for developers coming from C#.

## `==` → Compare Values

```python
a = [1, 2, 3]
b = [1, 2, 3]

print(a == b)
```

Output:

```text
True
```

Contents are equal.

---

## `is` → Compare Object Identity

```python
print(a is b)
```

Output:

```text
False
```

Because they are different objects in memory.

Think of:

```csharp
object.ReferenceEquals(a, b)
```

---

# Comparing with None

Use `is`, not `==`.

Recommended:

```python
customer = None

if customer is None:
    print("No customer")
```

Not recommended:

```python
if customer == None:
    ...
```

# Logical Operators with Comparisons

## AND

```python

print(age >= 18 and age <= 60)
```

Output:

```text
True
```

---

## OR

```python
country = "India"

print(country == "India" or country == "USA")
```

Output:

```text
True
```

---

## NOT

```python

print(not is_admin)
```

Output:

```text
True
```

# Comparison Operators in AI/ML

You will frequently see comparisons such as:

### Filter Data

```python
ages = [10, 20, 30, 40]

adults = [age for age in ages if age >= 18]

print(adults)
```

Output:

```text
[20, 30, 40]
```

### Check Model Accuracy

```python
accuracy = 0.92

if accuracy >= 0.90:
    print("Deploy model")
```

### Data Validation

```python
if len(text) == 0:
    print("Input cannot be empty")
```

## Important Interview Point

A very common Python interview question is:

```python
a = [1, 2, 3]
b = [1, 2, 3]

print(a == b)
print(a is b)
```

Output:

```text
True
False
```

* `==` compares **values/content**
* `is` compares **whether both variables reference the same object in memory**

This distinction becomes important when working with complex objects, data structures, and AI frameworks.
